# connection-checker

Небольшой веб‑сервис на Next.js для диагностики качества интернет‑соединения **из браузера**:

- HTTP/HTTPS задержка (RTT) и «потери» на уровне `fetch()`
- скорость загрузки (downlink) по HTTPS с публичного CDN
- доступность (reachability) выбранных хостов из браузера (CORS / no‑cors)

## Что важно понимать

- Это **не ICMP ping** и не «чистая» потеря пакетов L3/L4. Всё измеряется так, как это видит обычная вкладка браузера: TLS + HTTP + `fetch`.
- Часть проб выполняется в **no‑cors (opaque)**: JS не видит HTTP‑статус, но факт успеха/ошибки транспорта виден.
- Цифры зависят от маршрута, DNS, VPN/прокси, корпоративных фильтров, расширений и Wi‑Fi.

## Как запустить локально

Требования: Node.js + npm.

```bash
npm install
npm run dev
```

Откройте адрес из вывода dev‑сервера (обычно `http://localhost:3000`).

Сборка:

```powershell
npm run build
npm run start
```

## Как устроены измерения

### HTTP RTT и «потери»

- Делается серия последовательных запросов к **внешним** ротируемым URL.
- Затем выполняется burst‑серия (в текущей реализации тоже **последовательно**).
- «Потери» — доля запросов, которые не завершились успешно для страницы (таймаут/ошибка транспорта; для CORS также учитываются 4xx/5xx).

### Скорость загрузки (HTTPS downlink)

- GET по HTTPS к публичному CDN (jsDelivr).
- Фазы: warm‑up → крупный объект → серия файлов.
- Скорость считается по реальному размеру ответа (`arrayBuffer.byteLength`) и времени.

### Reachability (HTTP/S)

- Пробы к хостам выполняются **поочерёдно**, результаты появляются в таблице по мере выполнения.
- Для строк с CORS виден HTTP‑статус; для no‑cors — только факт успешного запроса.
